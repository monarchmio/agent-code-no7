# Agent Code No.7 — 架构设计文档

## 1. 系统概述

Agent Code No.7 是一个多智能体协作系统，基于腾讯 ClawPro 平台构建，采用 **"总指挥 + 专业特工"** 的分层架构。

## 2. 核心设计理念

### 2.1 特工局隐喻
- 每个 Agent 是一位拥有特定技能的"特工"
- 总指挥（Commander）负责任务拆解与调度
- 特工之间可以协作完成复杂任务
- 任务完成后由总指挥做质量把控

### 2.2 设计原则
- **专业化**：每位特工深耕一个领域，确保专业深度
- **模块化**：特工可独立运行，也可组合使用
- **可扩展**：新增特工只需注册配置，无需改动核心架构
- **容错性**：单个特工失败不影响整体任务

## 3. 架构分层

```
┌──────────────────────────────────────────┐
│             表现层 (Presentation)          │
│     ClawPro UI / API / 企业微信集成        │
├──────────────────────────────────────────┤
│            编排层 (Orchestration)          │
│   Agent-07 Commander + Workflow Engine    │
├──────────────────────────────────────────┤
│             能力层 (Capability)            │
│  Agent-01~06 专业特工 + Prompt Templates  │
├──────────────────────────────────────────┤

├──────────────────────────────────────────┤

│     RAG 知识库 / 向量数据库 / 模板库      │
└──────────────────────────────────────────┘
```

## 4. 工作流引擎

### 4.1 DAG 编排
- 任务以有向无环图（DAG）形式定义
- 支持串行、并行、条件分支
- 每个节点对应一个 Agent 动作

### 4.2 并行执行
- 无依赖关系的步骤自动并行执行
- 例如尽调场景中，法律/财务/商业调查同时进行
- 最终由总指挥汇总各并行分支结果

### 4.3 错误处理
- **重试策略**：步骤失败后自动重试（最多2次）
- **降级策略**：非关键步骤失败时跳过并标注
- **超时控制**：单步和全局超时保护

## 5. Prompt 工程策略

### 5.1 Chain-of-Thought (CoT)
- 复杂分析任务使用思维链推理
- 确保输出的逻辑可追溯性

### 5.2 ReAct (Reason + Act)
- 需要调用外部工具时使用
- 推理 → 行动 → 观察 循环

### 5.3 Reflection
- 特工输出经过自我反思优化
- 总指挥进行二次质量审核

## 6. 安全与合规

- **数据隔离**：不同客户数据严格隔离
- **敏感信息**：合同/财务数据仅在处理期间保留，处理后不持久化
- **免责声明**：所有输出标注"仅供参考"
- **审计日志**：记录所有特工操作日志
- **权限控制**：基于角色的访问控制（RBAC）
